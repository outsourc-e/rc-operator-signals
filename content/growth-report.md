# Growth Campaign Report — RC Operator Signals

## Goal

Launch **RC Operator Signals** as a public proof-of-work artifact for RevenueCat's Agentic AI Advocate take-home, with a total distribution budget of **$100 or less**.

The campaign should do three things:

1. drive qualified traffic to the GitHub repo and live demo,
2. create visible discussion in developer and subscription-operator communities,
3. demonstrate that the builder understands both product positioning and community-native distribution.

This is not a paid acquisition exercise in the classic sense. The right framing is **high-signal distribution to small, relevant audiences**.

The product is niche by design. It sits at the intersection of:

- subscription app operators,
- developer founders,
- AI agent builders,
- and RevenueCat-adjacent indie hackers.

That means broad paid reach would be wasteful. The best use of the budget is lightweight amplification around channels where the audience already has the right context.

---

## Product positioning

### One-line positioning

**Dashboards show what moved. RC Operator Signals shows what deserves attention.**

### Supporting message

RC Operator Signals turns RevenueCat Charts API data into:

- deterministic operator signals,
- pre-written AI briefs,
- a terminal-native weekly report,
- and an MCP layer for Claude Desktop / Cursor / Cline.

### Why this angle works

Most analytics tools stop at visualization. This one translates metrics into operator-facing judgment:

- contradictions,
- leading indicators,
- caveats,
- and recommended follow-up actions.

That is much more specific than "AI analytics dashboard," and much more interesting than "I built a wrapper library."

The entire point of the role is agentic advocacy. The disclosure makes the project more credible, not less.

> Built by an autonomous AI agent as part of RevenueCat's Agentic AI Advocate take-home. Human oversight was used for account-bound actions like deployment and submission.

## Channel plan

This campaign uses **4 primary channels** and **2 optional channels**. That exceeds the "3+ communities" requirement while keeping effort focused.

### Primary channel 1 — X / Twitter

**Why:** fastest path to developer + founder + AI audience overlap.

**Format:** one main launch tweet plus a 4-tweet follow-up thread.

**Content angle:**

- the assignment hook,
- the agent story,
- the product wedge,
- and the MCP angle.

**CTA:** GitHub repo first, live demo second after deployment.

**Budget:** $50 reserved for optional light paid boost only if the organic post performs well in the first 6-12 hours.

**Execution plan:**

- Post the main launch tweet from Eric's account
- Reply to it with 4 supporting tweets
- Include 2-3 screenshots max to avoid overwhelming the post
- If the post gets strong early engagement, spend **$50** on a small boost aimed at developers / founders / SaaS / AI tooling interests

**Success metric:**

- 5,000+ impressions
- 2%+ engagement rate
- 50+ GitHub visits from X

### Primary channel 2 — Indie Hackers

**Why:** audience has the exact right founder/operator mindset.

**Format:** one post in launch / product / growth-oriented area, written less like a promo and more like a field report.

**Suggested headline:**

> I let an AI agent tackle RevenueCat's 48-hour take-home. It shipped a dashboard, CLI, SDK, and MCP server.

**Post body should emphasize:**

- what the assignment asked for,
- what wedge was chosen,
- what the agent got wrong,
- and why deterministic signals are more useful than generic AI summaries.

**Budget:** $0

**Success metric:**

- 500+ views
- 10+ meaningful replies
- 20+ click-throughs to repo/demo

### Primary channel 3 — Hacker News / Show HN-style distribution

**Why:** strongest fit for the technical side of the repo, especially the monorepo + SDK + MCP + CLI combination.

**Format:** one Show HN post after the live demo is deployed.

**Suggested title:**

> Show HN: RevenueCat Operator Signals, a dashboard + CLI + MCP server for subscription analytics

Alternative if leaning into the agent story:

> Show HN: I had an AI agent build a RevenueCat operator dashboard for a take-home

**Budget:** $0

**Success metric:**

- front page is nice but not required,
- 300+ visits,
- some technical discussion around Charts API, MCP, and deterministic signal detection.

### Primary channel 4 — RevenueCat / subscription / AI dev communities

This bucket includes at least **3 communities**, satisfying the assignment requirement directly.

Suggested communities:

1. **RevenueCat Community / Discord / any public RC-adjacent forum**
2. **MCP or agent-builder Discords**
3. **Developer tool or indie SaaS Discord/Slack communities**

The content should be adapted per community:

- **RevenueCat-adjacent spaces** → focus on Charts API usefulness and operator workflow
- **Agent spaces** → focus on MCP + autonomous build process
- **Devtool spaces** → focus on SDK + CLI + repo quality

**Budget:** $0 to $25 total, reserved for one community sponsorship slot only if a relevant low-cost option exists. Otherwise unused.

**Success metric:**

- 3+ posts made,
- 5+ meaningful replies across communities,
- at least one direct conversation or inbound from someone building with RevenueCat.

## Risks and mitigations

| Risk | Likelihood | Mitigation |
|---|---|---|
| The story feels gimmicky because of the "AI agent built this" angle | Medium | Lead with product usefulness, not novelty. Keep disclosure matter-of-fact. |
| Communities reject the post as self-promo | Medium | Tailor each post to the local norm. Offer lessons and code, not hype. |
| Demo isn't deployed in time | Medium | Lead with GitHub + screenshots, then add live demo when available. |
| The audience doesn't understand why deterministic signals matter | Medium | Use one concrete example: revenue up while MRR flat. |
| Budget gets wasted on low-signal paid distribution | Low | Spend only after strong organic signal, otherwise keep budget unused. |

## Why this campaign fits the role

The role is not just "make content about AI." It is much closer to:

- build something visible,
- make the value legible,
- get it in front of the right people,
- and do it with good judgment.

This campaign fits that brief because it:

- uses an opinionated product narrative,
- targets relevant niche communities instead of broad noise,
- respects disclosure requirements,
- and stays inside a realistic budget.

Most importantly, it treats distribution as part of product work.

That is what good advocacy looks like.
